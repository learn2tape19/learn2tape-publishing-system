## Versioning Policy

### Major Version Changes (1.0 → 2.0)

Major version changes represent fundamental restructuring of the publishing system or curriculum architecture. These require evidence of material limitation from production or analytics.

### Minor Version Changes (1.0 → 1.1)

Minor version changes represent refinements, new operational documents, or expanded capabilities that do not alter the core architecture.

### Patch Version Changes (1.0.0 → 1.0.1)

Patch changes represent documentation corrections, metadata updates, or clarifications without operational impact.
